\section{Introduction}
In this assignment, we have used IMDB dataset
(\href{https://datasets.imdbws.com/}{https://datasets.imdbws.com/}) to populate our database and PostgreSQL as the RDBMS for piping in the data.

\section{List of DB Files}
\begin{itemize}
	\item \textbf{person} - Contains the person id, name, birth year and death year. 'Person ID' is the primary key. 
	\item \textbf{region} - Contains the name of region (primary key). 
	\item \textbf{genre} - Contains the type of genre as attribute which is also the primary key.
	\item \textbf{movie} - Contains MovieID, primary title, original title, isAdult, number of votes and Rating attributes. `MovieID' is the primary key.
	\item \textbf{moviegenre} - Contains the `MovieID' which is a foreign key referencing the `movie' table and `genre type' which references the `genre' table.
	\item \textbf{directorMovie} - Contains the `MovieID' which is a foreign key referencing the `movie' table and `PersonID' which references the `person' table to indicate the person who is director in the movie.
	\item \textbf{actormovie} - Contains the `MovieID' which is a foreign key referencing the `movie' table and `PersonID' which references the `person' table to indicate the person who is actor in the movie.
	\item \textbf{crewmovie} - Contains the `MovieID' which is a foreign key referencing the `movie' table and `PersonID' which references the `person' table to indicate the person who is a crew member in the movie. It also contains the `type' attribute to indicate the type of job like `2 for producer', `3 for writer', `4 for composer', etc.
	\item \textbf{regionmovie} - Contains the `MovieID' which is a foreign key referencing the `movie' table and `RegionName' referencing the 'region' table. It also contains the attribute of `runtime' (runtime of movies in each region) , `title' of movies in each region, `start date' and `end date'. MovieID, RegionName and Title combines to form primary key.
	\item \textbf{tvseries} - Contains TVID, primary title, original title, isAdult, number of votes and Rating attributes. `TVID' is the primary key.
	\item \textbf{tvgenre} - Contains the `TVID' which is a foreign key referencing the `tvseries' table and `genre type' which references the `genre' table.
	\item \textbf{directortv} - Contains the `TVID' which is a foreign key referencing the `tvseries' table and `PersonID' which references the `person' table to indicate the person who is director in the tvseries.
	\item \textbf{actortv} - Contains the `TVID' which is a foreign key referencing the `tvseries' table and `PersonID' which references the `person' table to indicate the person who is actor in the tvseries.
	\item \textbf{crewtv} - Contains the `TVID' which is a foreign key referencing the `tvseries' table and `PersonID' which references the `person' table to indicate the person who is a crew member in the tvseries. It also contains the `type' attribute to indicate the type of job like `2 for producer', `3 for writer', `4 for composer', etc.
	\item \textbf{episodes} - Contains EpisodeID, title, original title, season number, episode number, isAdult, number of votes and Rating attributes. `EpisodeID' is the primary key. `TVSeriesId' is the foreign key which references the `tvseries' table to associate itself to the parent tv-series.  
	\item \textbf{directorepisode} - Contains the `EpisodeID' which is a foreign key referencing the `episodes' table and `PersonID' which references the `person' table to indicate the person who is director in the specific episode.
	\item \textbf{actorepisode} - Contains the `EpisodeID' which is a foreign key referencing the `episodes' table and `PersonID' which references the `person' table to indicate the person who is actor in the specific episode.
	\item \textbf{crewepisode} - Contains the `EpisodeID' which is a foreign key referencing the `episodes' table and `PersonID' which references the `person' table to indicate the person who is a crew member in the episode. It also contains the 'type' attribute to indicate the type of job like `2 for producer', `3 for writer', `4 for composer', etc.
	\item \textbf{regionepsiode} - Contains the `EpsiodeID' which is a foreign key referencing the 'episodes' table and `RegionName' referencing the `region' table. It also contains the attribute of `runtime' (runtime of episodes in each region) , `start date' and `end date'. `EpisodeID' and `RegionName' combines to form primary key.
	
\end{itemize}
